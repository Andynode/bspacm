Detailed Description

Include file to define LEUART instances for EFM32 devices.

In your periph_config.c file you should define parameter macros described below, and include this file, once for each LEUART device you need in your application. Make sure the periph_config.c file also defines a pin mux assignment for the peripherals you will used.

Define:

  • BSPACM_INC_PERIPHNUM to the instance number that should be adjoined to the peripheral name (e.g. 0 for LEUART0, 1 for LEUART1)
  • BSPACM_INC_LOCATION to the LEUART_ROUTE_LOCx to use for GPIO configuration
  • BSPACM_INC_LFBSEL to the LFB CMU_Select_TypeDef value to use to as the clock source for LFB. Explicitly set to 0 to leave the existing LFB clock source in place; the default if not provided is cmuSelect_LFXO.
  • BSPACM_INC_TX_BUFFER_SIZE (optional) to the desired size for the internal transmit fifo
  • BSPACM_INC_RX_BUFFER_SIZE (optional) to the desired size for the internal receive fifo

This will add the following symbols (where # denotes the peripheral number):

  • LEUART::_IRQHandler which gets installed in the interrupt vector
  • xBSPACMdeviceEFM32periphLEUART# which may be used as a handle to operations described in <bspacm/periph/uart.h>.
